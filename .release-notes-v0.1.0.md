## syncthing-rust v0.1.0 - First Alpha Release

**Core capability verified**: Rust node can perform bidirectional file sync with official Go Syncthing.

### Verified Capabilities
- Push/Pull E2E: Rust <-> Go file push and pull, content checksum verified
- Full BEP protocol chain: TLS handshake -> Hello -> ClusterConfig -> Index -> Request/Response
- Filesystem watcher: `notify` crate real-time monitoring, 1s debounce -> scan -> IndexUpdate
- REST API: `/rest/system/status`, `/rest/system/connections`, `/rest/db/status`, device/folder CRUD
- TUI interactive interface: device/folder management, real-time logs, help page
- Config persistence: JSON config, `local_device_id` auto-saved
- Multi-transport architecture: TCP+TLS / WebSocket / Proxy / DERP Relay

### Test & Build
```
cargo test --workspace --lib  -> 257 passed, 0 failed, 1 ignored
cargo build --release         -> 0 errors, 0 warnings
```

### Known Limitations
- Long-term connection stability pending 72h stress test validation
- TUI and REST API runtime config not fully synchronized (consistent after restart)
- QUIC / full ICE hole punching not yet implemented

### Quick Start
```bash
git clone https://github.com/juice094/syncthing-rust.git
cd syncthing-rust
git checkout v0.1.0
cargo build --release -p syncthing
cargo run --release -p syncthing -- run
```
